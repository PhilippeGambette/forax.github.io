La syntaxe

Un style est une liste de sélecteurs auquels
sont associées des propriétés. 


selecteur1 {
  nom1: valeur1;
  nom2: valeur2;
}

/* ceci est un commentaire */
selecteur2 {
  autre-nom: autre-valeur;
}

'/*' démarre un commentaire, '*/' le fini

Cascading Style Sheet

Cascading signifie que si l'on applique un style à un élément, le style est propagé [1] à tous ses sous-éléments

[1] ou hérité si l'on regade du point de vue des fils.

CSS (algo 1/2)

L'algorithme marche en 2 temps
1. on applique les sélecteurs dans l'ordre du fichier 


body {
  color: blue;
}
li {
  color: green;
}

CSS (algo 2/2)

L'algorithme marche en 2 temps
2. on propage les styles 


body {
  color: blue;
}
li {
  color: green;
}

Les liens

Il existe des pseudo-classes reflétant l'état d'un lien 


A:link {  /* lien non visite */
  color: green;
}
A:visited { /* lien visite */
  color: yellow;
}
A:hover { /* souris au dessus du lien */
  color: teal;
}
A:active { /* on click sur le lien */
  color: red;
}

Propriété des boites

Inline vs block

Pour l'affichage, les éléments sont séparés en 2 catégories

  • Les inlines, qui comme du texte sont
    affichés de gauche à droite
  • Les blocks, (boites) prennent toute la largeur
    et dont la hauteur est de la somme des
    hauteurs des sous-éléments

     

    A et BOLD sont des inlines, P, UL ou LI sont des blocks

Span vs div

Balises sans propriété par défaut.

  • <span> est un inline
  • <div> est un block

CSS Box Model

Le CSS définie 3 propriétés définissant les boites autour du contenu

  • padding: espace à l'intérieur de la boite
  • border: bordure autour des boite
  • marging: espace entre les boites
